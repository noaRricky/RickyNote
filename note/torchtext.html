# Torchtext源码笔记

在使用pytorch处理自然语言处理(NLP)任务的时候，除了实现模型代码，如何对文本进行预处理，并且生成model需要的训练数据(Tensor)，在代码实现的过程中同样使我们头疼，因此facebook为我们提供与pytorch协同作战的文本预处理库torchtext

# torchtext包含功能

  • 分词(Word segmentation)
  • 生成单词库(Vocab)
  • 生成训练时使用的批数据的操作包括
    • 单词从文本到LongTensor转化
    • 添加开始字符(Start OS), 结束字符(End OS)
    • 将长度不同的文本序列通过padding操作对齐
  • 生成标准的pytorch模型训练需要的Dataset和Dataloader

# 使用torchtext

开源社区中有千千万万种开源库处理文本预处理工作，使用torchtext开作为我们文本预处理的基础库是因为它提供了一种针对pytorch模型文本预处理的代码规范，秉承着约定大于灵活的原则，我们只要按照它要求的规范编写代码就能写出易扩展、可读性较好的代码

然而不幸的是torchtext的官方文档十分简洁，只提供一些针对比较规范的数据文件的预处理方法，并且每个模块和类的注释不够详尽，因此在使用它的时候不得不去看源码，万幸的是源码可读性很好，可能这就是它文档写得不够清楚的原因吧。

本文会重点介绍torchtext每个模块的作用，以及在实际编写代码的时候，我们会经常使用的模块，以减少bug的产生。

# 根目录

python工程项目的基本结构，我们可以望文生义地理解每个文件夹的功能：

  • build-tools - 处理项目自动化构建部署(略)
  • docs - 项目相关文档(就我说的写得很略的那个)
  • examples - 使用用例(我就不懂官网为毛不把这个介绍清楚)
  • test - 测试文件夹(官方库果然测试少不了)
  • torchtext - 源码文件夹(划重点)

其余文件都是针对项目的配置，包括语法检查，单元测试，自动化部署，依赖包等内容，一股浓浓的高端规范项目之气扑面而来，但我们根本不需要知道这些文件具体干了什么

# 项目目录

  • data: 里面保存里文本预处理的所有基础功能模块(重点)
  • datasets: 里面存放针对不同自然语言处理任务的功能模块，它使用data中的模块，实际编写代码的时候我们可以仿照该模块的代码实现我们的预处理模型
  • experimenial: 实验模块(忽略)
  • utils.py: 功能文件
  • vocab.py: 存放单词集模型

# data文件夹

  • batch: Batch类定义，主要在iterator中输出minibatch时使用(略)
  • dataset: 存放继承了pytorch.utils.data.Dataset的类
    • Dataset: 定义了针对自然语言处理任务数据集的类(重点)
    • TabularDataset: 继承于Dataset,用于读取各种基础格式的数据集的对象, 但是当数据集不是标准格式时候无法使用
  • example: Example类定义(重点)
  • field:
    • RawField (定义了文本预处理的基本操作接口)
    • Field (文本预处理核心对象)
    • ReversibleField (继承自Field)
    • SubwordField (继承自Field)
    • NestedField (继承自Field)
  • functional (此文件包含实验功能, 略)
  • iterator:
    • Iterator: 基础的数据集迭代器类
    • BPTTIterator: 针对Language Modeling问题数据集的迭代器类
    • BucketIterator: 将长度相似的序列数据放在一个minibatch的迭代器类
