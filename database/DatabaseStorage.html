# 存储设备金字塔

  • 图中各个存储单元由上往下分别为CPU寄存器、CPU缓存、DRAM(动态随机存取存储器)、SSD(固态硬盘)、HDD(机械硬盘)、Network Storage(网络存储设备)
    • DRAM(Dynamic Random Access Memory): 动态随机存取存储器，计算机内存就是DRAM
  • 途中各个设备的读写速率从上至下，以此下降
  • Volatile(易失性设备)和Non-Valatile(非易失性设备)存储器器的区别
    • 易失性设备(Volatile Device)
      • 易失性意味着如果让机器断电，则数据会丢失
      • 易失性存储支持具有字节可寻址位置的快速随机访问
      • 在本文中，我们将该存储类称为“内存”。
    • 非易失性设备(Non-Volatile Device)
      • 非易失性意味着存储设备不需要连续提供电源用于设备保留它正在存储的数据
      • 传统上，非易失性存储在顺序访问方面更好 (可以一次性读取连续的数据块) 和被指针指向的数据
      • 在本文中，我们将该存储类称为“外存”

# 不同存储设备读写时间

实际读写时间 设备名称 相对读写时间
0.5 ns L1 Cache 0.5 sec
7 ns L2 Cache Ref 7 sec
100 ns DRAM 100 sec
150,000 ns SSD 1.7 days
10,000,000 ns HDD 16.5 weeks
~30,000,000 ns Network Storage 11.4 months
1,000,000 ns Tape Archives 31.7 years

上图中，左侧时间为真实机器读写时间，右侧将时间放大1000倍，帮助我们理解不同设备之间读写效率的巨大差异，从图中我们可以分析得出，应该尽量让数据库使用设备金字塔的上层，用来加速数据的读写。

# 读取数据为什么不使用操作系统的读取策略

首先，我们由下图来回顾操作系统，读取数据的过程

如图所示，首先用户选择读取page1，os从磁盘中读取数据到物理内存，然后选择读取page3，os将数据加载到物理内存后，当用户选择读取page2时，我们应该如何处理？同时，数据库中存储的数据是按照一定顺序存储了，os的读取方式没用充分利用该特性；导致了，数据读取和写入都很慢。该方法还会遇到更多问题：

  • 不能处理多个线程读取数据时候失败的情况
  • 这个方式可以很好解决数据读取问题，但是无法解决多个线程的数据修改过程

综上所述，操作系统并不是你的好朋友！

# 数据库设计目标

由上文，对不同存储设备的了解，我们可以得出数据库管理系统的设计目标如下:

  • 允许数据库管理系统处理超出系统内存的数据
  • 尽量使用上层设备，提高数据读写的效率
  • 以正确的顺序将脏页刷新到磁盘
  • 针对数据特征(连续性数据)的读存策略
  • 缓冲区更换政策
  • 线程/进程调度
